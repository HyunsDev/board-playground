# Error와 Exception의 구분

- 2025년 11월 28일

## 개선 배경

[48fe54e](48fe54e39c84d256cb838aaf2f422b7d25835e94) 전까지는 Exception과 Error이 명확한 구분 없이 사용되었다.

이로 인해, 내부 오류나 도메인 오류가 실제 백엔드와 클라이언트 사이에서 구분 없이 적용되었고, 개발에 혼란을 초래했다.

이에 따라 Exception과 Error를 명확하게 정의하여 혼란을 방지하고자 한다.

## 개선 방안

- **Error**는 실제 오류 그 자체.
- **Exception**은 백엔드와 프론트엔드 사이에서 주고 받는 응답 형식, 또는 그에 따른 기능을 정의한다.

즉 Exception은 사용자(또는 클라이언트)에게 표시될 수 있는 오류를 의미한다.
