# 커밋 메세지 규칙

## 기존 구조: `type(scope): description`

```plaintext
<타입>(<범위>): <제목>

<본문> (선택 사항)
<꼬리말> (선택 사항)
```

### type

- `feat`: 신규 기능 추가
- `fix`: 버그 수정
- `refactor`: 기능 변경 없는 코드 구조 개선
- `style`: 코드 포맷팅, 세미콜론 누락 수정, 린트 적용 등
- `docs`: 문서 수정
- `test`: 테스트 코드 추가/수정
- `chore`: 빌드, 패키지 매니저 설정 등의 자잘한 변경
- `ci`: CI 관련 추가/수정
